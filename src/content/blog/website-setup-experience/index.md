## 缘起

一直想要搭建一个属于自己的个人网站，既能展示我的技术能力，也能分享我的想法和项目。经过一番调研，最终选择了 Astro + Pure 主题的组合。这篇文章记录了我从零开始搭建网站的完整历程。

## 技术选择

### 为什么选择 Astro？

在选择技术栈时，我考虑了以下几个因素：

1. **性能优先**：Astro 的静态站点生成能力非常强大，能够生成极快的网站
2. **组件化开发**：支持多种框架的组件，灵活性很高
3. **学习成本**：相比其他框架，Astro 的学习曲线相对平缓
4. **生态丰富**：有大量的主题和插件可供选择

### Pure 主题的优势

Pure 主题吸引我的地方：

- **简洁美观**：设计风格简约现代，符合我的审美
- **功能完整**：内置搜索、RSS、SEO 等功能
- **高度可定制**：可以轻松修改样式和功能
- **文档完善**：有详细的中文文档

## 搭建过程

### 1. 环境准备

首先安装了 Node.js 和包管理器，然后克隆了 Pure 主题的模板：

```bash
git clone https://github.com/cworld1/astro-theme-pure.git
cd astro-theme-pure
npm install
```

### 2. 基础配置

修改了 `src/site.config.ts` 文件，配置了网站的基本信息：

- 网站标题和个人信息
- 导航菜单结构
- 社交媒体链接
- 主题颜色和样式

### 3. 内容定制

#### 删除不需要的功能

根据我的需求，删除了一些不需要的功能：

- **Docs 系统**：删除了整个文档系统，简化了网站结构
- **模板文章**：删除了所有示例文章，为我的内容腾出空间
- **多余组件**：移除了一些不常用的组件

#### 定制 Links 页面

将原来的友链页面改造成了推荐页面，包含三个分类：

- **推荐博主**：我关注的技术和生活博主
- **推荐游戏**：给我留下深刻印象的游戏
- **推荐书籍**：塑造我思维方式的书籍

### 4. 样式调整

#### 颜色主题

选择了深蓝色作为主色调，既专业又不失活力：

```css
--color-primary: #4f46e5;
--color-primary-light: #6366f1;
--color-primary-dark: #3730a3;
```

#### 字体和排版

使用了系统字体栈，确保在不同设备上都有良好的显示效果：

```css
font-family: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif;
```

### 5. 内容创作

#### 博客文章

创建了这篇搭建心得的文章，记录整个过程。文章结构包括：

- 技术选择的思考过程
- 具体的搭建步骤
- 遇到的问题和解决方案
- 最终的成果展示

#### 项目展示

准备了一些个人项目的展示，包括：

- 开源项目贡献
- 技术实验和探索
- 学习笔记和总结

## 遇到的问题和解决方案

### 1. 文档系统删除

删除 docs 系统时遇到了一些引用问题，需要：

- 删除 `src/content/docs` 目录
- 删除 `src/pages/docs` 目录
- 修改 `content.config.ts` 文件
- 更新相关组件的引用

### 2. 样式定制

在定制样式时遇到了一些挑战：

- **主题色调整**：需要理解 CSS 变量的作用域
- **组件样式**：某些组件的样式需要深度定制
- **响应式设计**：确保在不同设备上都有良好的显示效果

### 3. 内容管理

内容管理方面的一些考虑：

- **文章结构**：设计了清晰的文章分类和标签系统
- **图片优化**：使用 Astro 的图片优化功能
- **SEO 优化**：添加了必要的 meta 标签和结构化数据

最终使用的技术栈：

- **框架**：Astro 4.x
- **主题**：Astro Theme Pure
- **样式**：UnoCSS
- **部署**：Vercel
- **搜索**：Pagefind

- **数学公式**：KaTeX
